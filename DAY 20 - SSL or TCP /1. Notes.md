# SSL/TLS (Secure Sockets Layer / Transport Layer Security)

## What is SSL/TLS?

SSL/TLS is a security protocol that provides **secure communication** between a client and a server over a network.

It ensures:

* Encryption
* Authentication
* Data Integrity

**Note:** SSL is deprecated. Modern applications use **TLS**, but the term **SSL** is still widely used.

---

# Why Do We Need SSL/TLS?

Suppose a user accesses a website.

```
User
   │
HTTP GET Request
   │
────────────►
Server
```

The server asks:

```
Who are you?
```

The user sends:

```
Username
Password
```

Server authenticates the user.

Everything works.

---

## Problem with HTTP

Imagine there is a hacker between the client and server.

```
User
   │
Username
Password
   │
──────► Hacker ──────► Server
```

The hacker can read:

* Username
* Password
* Credit Card
* Banking Details

because HTTP sends data in **plain text**.

---

# Solution?

Encrypt the data before sending.

But now another problem appears.

---

# Symmetric Encryption

## Definition

Symmetric Encryption uses **one shared key**.

The same key is used for:

* Encryption
* Decryption

---

## Working

```
User

Secret Key
      │
Encrypt Data
      │
────────────►

Server

Same Secret Key

Decrypt Data
```

---

## Advantages

* Very Fast
* Less CPU Usage
* Best for large data

---

## Disadvantage

How do we securely send the secret key?

```
User

Secret Key
     │
────────────►

Hacker copies key

────────────►

Server
```

Now hacker has

* Data
* Secret Key

Communication is compromised.

---

# Asymmetric Encryption

To solve this problem, we use **Public Key Cryptography**.

Instead of one key,

we have

```
Public Key

Private Key
```

---

## Public Key

* Shared publicly
* Used for Encryption
* Used for Signature Verification

---

## Private Key

* Never Shared
* Stored Securely
* Used for Decryption
* Used for Digital Signatures

---

# How are these keys generated?

Using

```bash
ssh-keygen
```

or

```bash
openssl
```

depending on the use case.

---

# SSH Example

Generate keys

```bash
ssh-keygen
```

Creates

```
id_rsa
```

Private Key

and

```
id_rsa.pub
```

Public Key

---

## SSH Authentication

```
Client

Private Key
      │
      │
────────────►

Server

authorized_keys

contains

Public Key
```

Server verifies

Private Key

using

Public Key

Access Granted

---

# Why is this secure?

Only the owner has

```
Private Key
```

Public Key alone cannot be used to decrypt data.

---

# Problem with Asymmetric Encryption Alone

Suppose we encrypt **all communication**

using RSA.

```
2 GB File

↓

RSA Encryption
```

Very slow.

Consumes lots of CPU.

Therefore

SSL/TLS uses **both**

* Asymmetric Encryption
* Symmetric Encryption

---

# SSL/TLS Uses Hybrid Encryption

The server generates

```
Public Key

Private Key
```

using

```bash
openssl
```

---

The client generates

```
Session Key

(AES Key)
```

which is

Symmetric.

---

# SSL/TLS Handshake

```
Client
      │
1. HTTPS Request
      │
────────────►

Server

2. Sends Public Key
(or Certificate)

◄────────────
```

---

Client now has

Server Public Key

Client creates

```
AES Session Key
```

Encrypts it

using

Server Public Key

```
AES Key

↓

Encrypted

↓

────────────►

Server
```

---

Server uses

Private Key

to decrypt

```
Encrypted AES Key

↓

AES Session Key
```

Now

Both Client

and

Server

have

the same AES Session Key.

---

# Secure Communication Starts

```
Client

AES Key

↓

Encrypted Data

↓

────────────►

Server

AES Key

↓

Decrypt Data
```

All communication now uses

AES

because

AES is very fast.

---

# Why Can't Hacker Read the Data?

```
User

Encrypted AES Key

↓

────────────►

Hacker

Copies Data

BUT

does NOT have

Server Private Key

↓

Cannot Decrypt
```

Hence

communication remains secure.

---

# Still One Problem...

How does the client know

the Public Key

really belongs

to the server?

What if

the hacker sends

its own Public Key?

This is called a

**Man-in-the-Middle Attack (MITM)**.

---

# Solution

Certificates

Instead of sending

only

```
Public Key
```

Server sends

```
Digital Certificate
```

---

# What is a Certificate?

A Digital Certificate contains

* Domain Name
* Organization
* Public Key
* Expiry Date
* Certificate Authority
* Digital Signature

---

Browser verifies

```
Is Certificate Valid?

Is Domain Correct?

Is Certificate Expired?

Is Certificate Trusted?
```

Only then

Secure Connection

is established.

---

# Certificate Authority (CA)

A CA is a trusted organization

that validates

the server identity.

Examples

* Let's Encrypt
* DigiCert
* GlobalSign
* Sectigo

---

# Certificate Creation Process

```
Server

↓

Generate Private Key

↓

Generate CSR

↓

Certificate Signing Request

↓

Send to CA

↓

CA Validates Domain

↓

Signs Certificate

↓

Returns Signed Certificate
```

---

# CSR (Certificate Signing Request)

CSR contains

* Domain Name
* Organization
* Public Key

It is sent

to

Certificate Authority

for signing.

---

# Browser Trust

When browser receives

Certificate

it checks

```
Certificate

↓

Trusted CA?

↓

Domain Match?

↓

Expiry?

↓

Signature?

↓

YES

↓

Secure HTTPS Connection
```

# HTTP vs HTTPS

| -------------- | -------------------- |

# Symmetric vs Asymmetric

| Symmetric            | Asymmetric            |
| -------------------- | --------------------- |
| One Key              | Public + Private Keys |
| Fast                 | Slower                |
| Used after Handshake | Used during Handshake |
| AES                  | RSA, ECC              |

# OpenSSL Utility

## What is OpenSSL?

OpenSSL is an open-source command-line tool used to:

* Generate Private Keys
* Generate Public Keys
* Generate CSR
* Generate Self-Signed Certificates
* View Certificate Details
* Verify Certificates
* Test SSL Connections
* Convert Certificate Formats

---

## Common Commands

Generate Private Key

```bash
openssl genrsa -out server.key 2048
```

Generate CSR

```bash
openssl req -new \
-key server.key \
-out server.csr
```

Generate Self Signed Certificate

```bash
openssl req -x509 \
-newkey rsa:2048 \
-keyout server.key \
-out server.crt
```

View Certificate

```bash
openssl x509 -text -noout \
-in server.crt
```

Check Expiry

```bash
openssl x509 \
-enddate \
-noout \
-in server.crt
```

Test HTTPS

```bash
openssl s_client \
-connect google.com:443
```

# Quick Revision

| Concept               | Key Point                           |
| --------------------- | ----------------------------------- |
| HTTP                  | Plain Text                          |
| HTTPS                 | Encrypted Communication             |
| Symmetric Encryption  | One Shared Key (AES)                |
| Asymmetric Encryption | Public & Private Key Pair (RSA/ECC) |
| Session Key           | Temporary AES Key                   |
| Digital Certificate   | Verifies Server Identity            |
| CSR                   | Certificate Signing Request         |
| CA                    | Signs Certificates                  |
| OpenSSL               | SSL/TLS Utility                     |
| SSL/TLS               | Hybrid Encryption (RSA + AES)       |

**Memory Trick:**

- **Handshake = RSA/ECC (Authentication + Secure Key Exchange)** 
- **Communication = AES (Fast Data Encryption)** 
- **Certificate = Trust** 
- **CA = Verifies Identity** 
- **OpenSSL = Certificate Toolbox**
