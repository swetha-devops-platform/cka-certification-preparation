# Node Selector

## What is a Node Selector?

**Node Selector** is the simplest scheduling mechanism in Kubernetes that allows you to schedule a Pod onto a specific node based on **node labels**.

Think of it as:

- **Pod:** "I want to run only on a node that has this label."

- Node Selector is a **Pod property** that matches **Node labels**.

---

# Why do we use Node Selector?

Node Selector is mainly used to:

* Run workloads on specific nodes
* Separate production and development workloads
* Schedule GPU workloads on GPU nodes
* Run database Pods on dedicated nodes
* Control Pod placement

---

# Real-Time Example

Suppose you have a Kubernetes cluster with 3 nodes.

```text
Control Plane
Worker-1
Worker-2
```

Label Worker-2:

```text
gpu=true
```

Now create a Pod with:

```yaml
nodeSelector:
  gpu: "true"
```

Result:

```
Scheduler
      ↓
Checks labels
      ↓
Worker-2 has gpu=true
      ↓
Pod Scheduled on Worker-2
```

# How Node Selector Works

Step 1

Node has labels

```text
gpu=true
```

↓

Pod created

↓

Scheduler checks

```yaml
nodeSelector:
  gpu: "true"
```

↓

Label matches

↓

Pod Scheduled

---

# Syntax

```yaml
spec:
  nodeSelector:
    <label-key>: <label-value>
```

Example

```yaml
spec:
  nodeSelector:
    environment: production
```

# Lab Example

## Step 1: Label a Node

Check nodes

```bash
kubectl get nodes
```

Add label

```bash
kubectl label node minikube-m02 gpu=true
```

Verify

```bash
kubectl get nodes --show-labels
```

Output

```text
NAME           LABELS
minikube       ...
minikube-m02   gpu=true
```

---

## Step 2: Create Pod

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  nodeSelector:
    gpu: "true"

  containers:
  - name: nginx
    image: nginx
```

Apply

```bash
kubectl apply -f pod.yaml
```

---

## Step 3: Verify

```bash
kubectl get pods -o wide
```

Output

```text
NAME     NODE
nginx    minikube-m02
```

# If Label Doesn't Exist

Suppose Pod has

```yaml
nodeSelector:
  gpu: "true"
```

but

```text
No node has gpu=true
```

Result

```text
Pod Status

Pending
```

Scheduler Event

```text
0/2 nodes are available:
2 node(s) didn't match Pod's node selector.
```

# Node Selector vs Node Name

| Node Selector                   | nodeName                                           |
| ------------------------------- | -------------------------------------------------- |
| Uses Labels                     | Uses Exact Node Name                               |
| Scheduler selects matching node | Scheduler is bypassed and Pod is assigned directly |
| Flexible                        | Static                                             |
| Recommended                     | Mostly used for testing and CKA tasks              |

---

# Node Selector vs Node Affinity

| Node Selector  | Node Affinity                |
| -------------- | ---------------------------- |
| Simple         | Advanced                     |
| Exact Match    | Expressions Supported        |
| AND logic only | AND & OR logic               |
| No preferences | Preferred and Required rules |

---

# Node Selector vs Taints & Tolerations

| Node Selector      | Taints & Tolerations  |
| ------------------ | --------------------- |
| Pod chooses Nodes  | Node rejects Pods     |
| Uses Labels        | Uses Taints           |
| Controls Placement | Controls Restrictions |

---

# Can Node Selector Work with Taints?

Yes.

Node

```text
gpu=true
```

and

```text
gpu=true:NoSchedule
```

Pod

```yaml
nodeSelector:
  gpu: "true"

tolerations:
- key: "gpu"
  operator: Equal
  value: "true"
  effect: NoSchedule
```

Result

```
Pod Scheduled
```

Without the toleration

```
Pending
```

because the node has a taint.

# Scheduler Flow

```
Pod Created
      ↓

      ↓

      ↓

      ↓
Check resources
      ↓

```

# CKA Exam Tips

* Node Selector uses **node labels** to schedule Pods.
* Labels are **key-value pairs** attached to nodes.
* The Pod is scheduled **only if all specified labels match**.
* If no node matches the labels, the Pod remains **Pending**.
* Node Selector supports **exact matching only**.
* Use `kubectl label node` to add labels.
* Use `kubectl get nodes --show-labels` to verify labels.
* Use `kubectl describe pod` to troubleshoot scheduling issues.
