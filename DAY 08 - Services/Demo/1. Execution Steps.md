**Step 05 — Login to Deployment on k8s (via Minikube SSH)**

Access application internally using curl

**Command : curl -L http://<ipaddress>:<port>/<instancename>**

This works inside the cluster network only. The application (e.g. Golang) cannot be accessed from outside the cluster.

---

**Step 06 — Create Service YAML**

vi service.yml

**Key Rule:**

- Labels and Selectors must be the same in both the Service and the Deployment YAML.

- The name in the pod template must also match — this is how the Service finds the right pods.

---
```
yaml# Inside the Service yaml
spec:
  selector:
    app: <same-label-as-deployment>   # Must match deployment labels exactly
  type: NodePort
  ports:
    - port: 80
      targetPort: <port-app-is-running-on>
```


---


**Step 07 — Create the Service**

- Command : kubectl apply -f service.yml

- Access via NodePort: curl -L http://$(minikube ip):<NodePort> Or open in browser http://minikube-ip:NodePort

- In production (EC2 on AWS): The WorkerNode IP address will be the EC2 instance IP.



**Important Trick**

- To get more info about API calls: - kubectl get pods -v=9 - 9 is the maximum verbosity level

- targetPort = the port where the application is actually running inside the container.

- Check background activity: kubectl get pods -v=9
