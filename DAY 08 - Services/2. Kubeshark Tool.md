# Kubeshark

## What is Kubeshark?

Kubeshark is an **open-source Kubernetes network observability and API traffic monitoring tool**. It is often called the **Wireshark for Kubernetes** because it captures, monitors, and analyzes traffic between Pods and Services inside a Kubernetes cluster.

---

## Why Kubeshark?

In a microservices architecture:

```text
Frontend Pod
      |
      v
Backend Pod
      |
      v
Database Pod
```

When communication fails:

* Which service made the request?
* What data was sent?
* What response was received?
* Which service is causing latency?

Kubeshark helps answer these questions.

---

## Key Features

### 1. Real-Time Traffic Monitoring

Monitor live traffic between Pods.

```text
Frontend --> Backend
Backend --> Database
```

---

### 2. API Inspection

Supports:

* HTTP
* HTTPS (limited)
* REST APIs
* GraphQL
* gRPC
* DNS

You can inspect:

* Request headers
* Request body
* Response body
* Status codes

---

### 3. Kubernetes Native

Works directly with:

* Pods
* Deployments
* Services
* Namespaces

No code changes required.

---

### 4. Troubleshooting

Useful for:

* API debugging
* Network troubleshooting
* Service communication validation
* Performance analysis

```text
+------------------+

+------------------+
         |
         v
+------------------+

+------------------+
         |
         v
+------------------+
| Worker Pods      |
+------------------+
         |
         v
Kubernetes Cluster
```

### Components

#### Hub

* Central management component
* Receives captured traffic
* Stores and processes data

#### Workers

* Run on cluster nodes
* Capture network packets
* Forward data to Hub

#### Frontend/UI

* Web dashboard
* Displays traffic and API calls

## Installation

### Install CLI

Linux:

```bash
curl -Lo kubeshark https://github.com/kubeshark/kubeshark/releases/latest/download/kubeshark_linux_amd64
chmod +x kubeshark
sudo mv kubeshark /usr/local/bin/
```

Verify:

```bash
kubeshark version
```

---

### Start Kubeshark

```bash
kubeshark tap
```

This command:

1. Deploys Kubeshark components.
2. Starts traffic capture.
3. Opens the web UI.

---

## Common Commands

### Start Monitoring

```bash
kubeshark tap
```

### Stop Monitoring

```bash
kubeshark clean
```

### View Version

```bash
kubeshark version
```

### Capture Specific Namespace

```bash
kubeshark tap -n production
```

## Example Use Case

Suppose:

```text
User Login Fails
```

Application logs show nothing.

Using Kubeshark:

1. Open UI
2. Find login API request
3. Check request payload
4. Check response code
5. Identify failing microservice

## Kubeshark vs Wireshark

| Feature               | Kubeshark | Wireshark |
| --------------------- | --------- | --------- |
| Kubernetes Aware      | Yes       | No        |
| Pod Visibility        | Yes       | No        |
| Service Visibility    | Yes       | No        |
| API Decoding          | Yes       | Limited   |
| Microservice Analysis | Excellent | Difficult |
